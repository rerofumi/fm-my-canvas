# fm-my-canvas

[日本語](README.md)

A desktop application for real-time prototyping of HTML/CSS/JS through chat-based interaction with LLMs.

Built with the motivation of bringing the ChatGPT Artifacts / Claude Artifacts experience to local LLMs. Connect to Ollama or OpenRouter and prototype UIs through conversation.

## Features

- **LLM Chat Interface** — Streaming chat with Ollama (local) or OpenRouter (cloud)
- **Real-time Artifact Preview** — Instantly preview HTML/CSS/JS generated by the LLM in an iframe sandbox
- **File Tree & Code Viewer** — Browse generated files with syntax highlighting
- **Agent Mode (Tool Calls)** — Advanced mode where the LLM directly manipulates files using tools like `read_file`, `write_file`, `apply_edit`, and `search_code`
- **Console Pane** — Real-time view of `console.*` output from both the app and the preview iframe
- **Session Management** — Create, switch, and delete multiple chat sessions; history persists across app restarts
- **Resizable Panels** — Drag to resize the Artifact pane; sidebar can be collapsed
- **Send Cancellation** — Stop button to cancel at any time during streaming or tool call loops

## Two Operating Modes

### Markdown Mode (Default)

The LLM outputs code in Markdown code blocks (`` ```html path=index.html ``), and the app automatically parses, saves, and previews the files. Suited for simple prototyping.

### Agent Mode

The LLM uses Tool Call functionality to directly manipulate workspace files. Supports diff-based editing (`apply_edit`), reading existing code (`read_file`), code search (`search_code`), and more. Ideal for large-scale code operations and iterative modifications.

| Mode | Artifact Update | Best For |
|------|----------------|----------|
| Markdown | Auto-parse after LLM response | Creating new prototypes |
| Agent | Direct file operations via Tools | Modifying / refactoring existing code |

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Backend | Go 1.23 + [Wails v2](https://wails.io/) |
| Frontend | Svelte 5 + TypeScript + Vite 6 |
| LLM Providers | Ollama API / OpenRouter API (SSE streaming) |
| Data Persistence | JSON files (sessions, settings) |
| Artifact Server | Go standard `net/http` (ephemeral server on `127.0.0.1` random port) |

## Architecture

```
User → Svelte UI → Wails Binding → Go ChatService → Provider (Ollama/OpenRouter)
                                                      ↓ SSE Stream
                                          EventsEmit("llm-event") → Svelte UI update
                                          ArtifactManager → file writes
                                          ArtifactServer → iframe preview
```

- **Provider Interface** (`provider/`) — Unified handling of Ollama / OpenRouter via `Stream()` / `StreamWithTools()`
- **ArtifactManager** (`artifacts/`) — Per-session workspace management, atomic file writes, code search
- **ArtifactServer** (`artifacts/`) — HTTP server with `Cache-Control: no-store` to always serve the latest files
- **SessionManager** (`session/`) — UUID-based sessions with JSON file persistence
- **ToolManager** (`tools/`) — Registers and executes Agent mode tools (with 30s timeout)

## Setup

Prerequisite: [mise](https://mise.jdx.dev/) must be installed.

```powershell
# Install tools (Go, Node.js)
mise install

# Install Wails CLI
mise run setup

# Install frontend dependencies
mise run frontend:install
```

## Development

```powershell
# Start dev server (Hot Reload)
mise run dev
```

## Build

```powershell
# Production build
mise run build
```

The build output is generated in `build/bin/`.

## Testing

```powershell
# Run all tests
mise run test

# Verbose output
mise run test:verbose
```

## Settings

The following can be configured from the Settings screen in the app. Settings are saved to `~/.config/fm-my-canvas/config.json`.

| Item | Description | Default |
|------|-------------|---------|
| Provider | `ollama` or `openrouter` | `ollama` |
| Ollama Endpoint | Ollama API endpoint | `http://localhost:11434` |
| Ollama Model | Model name to use | `llama3` |
| OpenRouter API Key | OpenRouter API key | (empty) |
| OpenRouter Model | Model to use on OpenRouter | (empty) |
| Agent Mode | Enable Tool Call-based Agent mode | OFF |

## Usage

1. Click **+ New** in the sidebar to create a new chat session
2. Type a prompt in the chat input and press **Ctrl+Enter** or **Send**
3. When the LLM outputs code, files automatically appear in the Artifact pane
4. Use the **Preview** tab in the Artifact pane for live preview, **Code** tab for per-file source code, **Console** tab for log output
5. Give follow-up instructions to update code (conversation context is maintained within the session)
6. Enable Agent Mode for precise file operations via Tool Calls

## Available Tools in Agent Mode

| Tool | Description |
|------|-------------|
| `read_file(path)` | Read a file in the workspace |
| `write_file(path, content)` | Write a file to the workspace |
| `list_files([path])` | List files in the workspace |
| `apply_edit(path, search, replace)` | Apply a search/replace edit to a file |
| `search_code(pattern, [file_pattern])` | Search code with regex |

Tool call rounds are limited to 10 per message, with a 5-minute overall timeout. Individual tool execution has a 30-second timeout.

## Data Storage

| Data | Path |
|------|------|
| Settings | `~/.config/fm-my-canvas/config.json` |
| Session history | `~/.config/fm-my-canvas/sessions/<uuid>.json` |
| Artifact files | `~/.config/fm-my-canvas/artifacts/<session-uuid>/` |

## Security

- Generated scripts run inside an `<iframe>` with the `sandbox` attribute, preventing unauthorized access to the local filesystem
- All file operations by tools are restricted to the workspace via path validation (path traversal prevention)
- File size is limited to 1MB for both reads and writes
- The artifact server listens only on `127.0.0.1` with a random port
