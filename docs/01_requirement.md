# 01. 要求仕様書 (Requirement Specification)

## 1. 概要
本アプリは、LLM（Ollama / OpenRouter）との対話を通じて、フロントエンドの Canvas（HTML/CSS/JS）をプロトタイピング・生成するためのデスクトップアプリケーションです。生成物を即座にプレビューでき、最終的に独立したファイル群としてエクスポートできることを目的とします。

## 2. ユーザーストーリーとユースケース

### 2.1. LLM プロバイダの管理
*   **US-01 プロバイダの選択**: ユーザは利用する LLM プロバイダ（Ollama または OpenRouter）をアプリケーションの設定画面、あるいはチャット開始前に選択できる。
*   **US-02 エンドポイントと認証キーの設定**: ユーザは OpenRouter の API キーや、ローカル Ollama のエンドポイント（ホスト・ポート）を保存・管理できる。これらの情報は暗号化または適切に隠蔽されて保存される。

### 2.2. チャットセッションとプロンプト入力
*   **US-03 プロンプトの投入**: ユーザはテキスト入力欄に、作成したい Canvas/UI の要件やデザイン指示を入力できる。
*   **US-04 会話コンテキストの維持**: ユーザは過去のやり取りを踏まえた修正指示（例：「背景を暗くして」「ボタンを中央に寄せて」）を継続的に行える。アプリケーションはこれまでの会話履歴をセッションとして保存・管理する。
*   **US-05 複数チャットの管理**: ユーザは複数のプロジェクト（セッション）を独立して管理でき、過去の履歴を任意に呼び出して再開できる。

### 2.3. Canvas / Artifact の生成とプレビュー
*   **US-06 生成ストリームの監視**: ユーザは LLM が生成する HTML、CSS、JavaScript ファイルの内容を、生成途中からリアルタイムに確認できる。
*   **US-07 動的プレビュー（Live Preview）**: 生成されたコードは、即座にアプリケーション内に組み込まれた独立したプレビュー領域（Sandbox 環境）でレンダリングされる。
*   **US-08 マルチファイル対応**: アプリケーションは HTML ファイルだけでなく、別ファイルとして参照される CSS や JS を正しく解決し、ブラウザと同様の挙動でファイルをバンドルして表示できる。
*   **US-09 結果の確認と刷新**: プレビュー画面において、ユーザーはリロードや、特定のコードブロックが更新された際に手動/自動で表示を更新できる。

### 2.4. 成果物のエクスポートと保存
*   **US-10 ファイル形式での保存**: ユーザは生成された Canvas を物理的なファイル（index.html, style.css, main.js などを含むディレクトリ構造）として、ローカルディスク上の任意の場所に保存できる。
*   **US-11 アーカイブのエクスポート**: ユーザは成果物全体を 1 つの ZIP ファイル等にまとめて一括エクスポートでき、外部環境や他のメンバーと簡単に共有できる。

### 2.5. アプリケーション設定と環境管理
*   **US-12 モデルの選択**: ユーザは Ollama のローカルモデル（例: llama3, qwen など）や、OpenRouter 経由で利用するモデルを柔軟に切り替えられる。
*   **US-13 テーマと UI カスタマイズ**: ユーザはダークモードや UI の文字サイズなど、開発時の視認性を高めるための基本的な表示設定をカスタマイズできる。

## 3. ノンファンクショナルな要件
*   **安全性**: 生成されるスクリプトはサンドボックス環境で実行され、ローカルのファイルシステムやネットワークに不適切にアクセスしないよう隔離されていること。
*   **応答性**: コードが生成される間も UI がフリーズせず、ストリーミングされる文字が即座に反映されるような低遅延なインターフェースを維持すること。
*   **可搬性**: エクスポートされた成果物は、特殊な開発環境なしに標準的なブラウザで動作し、単独の静的ファイルとして動作すること。
